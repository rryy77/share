# share — 画面共有専用アプリ

音ズレ・ラグを極力なくすことに特化した、**ホスト1名・参加者1名**の画面共有Webアプリ。
ZoomやLINEと違い、**映像・音声はサーバーを経由せず WebRTC で P2P 直送**するため、
トランスコードや大きなバッファリングが入らず、遅延と音ズレを最小化します。

- **シグナリング**（相手を見つける処理）… PeerJS クラウドを利用（サーバー構築不要）
- **映像・音声**… WebRTC で端末同士を直結（P2P）。サーバーは通らない
- **ホスティング**… Vercel に静的サイトとして配置するだけ

## 使い方

1. ホストがトップページ（`/`）を開く
2. **「画面共有を開始」** → 共有したいウィンドウ／画面全体を選択
   - 音声も共有したい場合は、選択ダイアログで「システム音声を共有」等にチェック
3. **「招待リンクを作成」** → 表示されたリンクを参加者に送る
4. 参加者はリンクを開き **「視聴を開始」** → ホストの画面が**全画面**で表示される

## ローカルで試す

```bash
npm start
# → http://localhost:3000 を開く
```

> 画面共有APIは「安全なコンテキスト」(HTTPS または localhost) でのみ動作します。
> ローカルでは必ず `http://localhost:3000` 経由で開いてください（`file://` では動きません）。

同じPC内なら、ホストを `http://localhost:3000`、参加者を別ウィンドウで招待リンクから開けば動作確認できます。

## Vercel へのデプロイ

このリポジトリはそのままデプロイできます（ビルド不要・静的配信）。

1. Vercel で「New Project」→ この GitHub リポジトリ（`share`）を Import
2. Framework Preset は **Other**（設定不要、`vercel.json` が効きます）
3. Deploy → 払い出された `https://xxx.vercel.app` を開けば本番稼働

招待リンクはアクセス中のドメインで自動生成されるため、本番URLで開けば本番URLのリンクになります。

## 構成

| ファイル | 役割 |
|---|---|
| `public/index.html` / `host.js` | ホスト画面 |
| `public/view.html` / `view.js`  | 参加者の全画面ビュー |
| `public/peer.js` | WebRTC共通設定・低遅延チューニング |
| `public/style.css` | スタイル |
| `vercel.json` | Vercel の静的ルーティング |
| `server.js` | ローカル開発用の静的サーバー（本番では未使用） |

## 低遅延・音ズレ対策（実装ポイント）

- **P2P直結** … 映像音声はサーバーを通らず端末間を直接流れる
- **映像と音声を同じ接続に載せる** … RTPタイムスタンプで本質的に同期＝音ズレが出ない
- **受信側バッファ最小化**（`playoutDelayHint = 0`）
- **送信ビットレート 12Mbps・解像度維持**（`maintain-resolution`）で文字までくっきり
- **音声処理オフ**（エコーキャンセル等を無効化）で加工遅延を回避

## 既知の制限・将来の改善

- **PeerJS の無料クラウド**を利用しているため、稀に接続が不安定になることがあります。
  安定運用したい場合は、自前の [PeerServer](https://github.com/peers/peerjs-server) を立てて
  `public/peer.js` の `PEER_CONFIG` に `host`/`port`/`path` を追記してください。
- **厳しいNAT・企業ネットワーク**では P2P が張れないことがあります。その場合は
  TURN サーバーを `PEER_CONFIG.config.iceServers` に追記する必要があります。
